# Java 泛型深度解析

## 一、概述

泛型（Generics）是 JDK 5 引入的特性，允许在编译时指定类型参数，提供编译时类型检查和消除类型转换。理解泛型是阅读源码、设计通用框架的基础。

**核心要点：**

- **类型擦除**：泛型只在编译期有效，运行时类型信息被擦除
- **通配符**：`?` 用于表示未知类型，配合 `extends`/`super` 限制边界
- **PECS 原则**：Producer Extends, Consumer Super，指导通配符使用
- **泛型局限**：不能实例化泛型数组、不能使用基本类型、不能实例化类型参数

## 二、泛型的本质

### 2.1 为什么需要泛型？

**泛型之前的代码**：

```java
List list = new ArrayList();
list.add("hello");
list.add(123);  // 编译通过，运行时可能出错

String s = (String) list.get(0);  // 必须强制类型转换
String s2 = (String) list.get(1); // ClassCastException！
```

**使用泛型后**：

```java
List<String> list = new ArrayList<>();
list.add("hello");
// list.add(123);  // 编译错误，类型安全

String s = list.get(0);  // 无需类型转换
```

**泛型的作用**：

| 作用 | 说明 |
|------|------|
| 类型安全 | 编译期检查类型，避免运行时 ClassCastException |
| 消除转换 | 无需手动强制类型转换 |
| 代码复用 | 一套代码适用于多种类型 |

### 2.2 类型擦除

**核心概念**：Java 泛型是**编译时语法糖**，运行时类型信息被擦除。

```java
// 源代码
List<String> strings = new ArrayList<>();
List<Integer> integers = new ArrayList<>();

// 编译后（字节码层面）
List strings = new ArrayList();
List integers = new ArrayList();

// 运行时验证
System.out.println(strings.getClass() == integers.getClass());  // true！
```

**擦除规则**：

| 泛型类型 | 擦除后类型 |
|---------|-----------|
| `<T>` | `Object` |
| `<T extends Number>` | `Number` |
| `<T extends Comparable<T>>` | `Comparable` |
| `<T extends Number & Comparable>` | `Number`（第一个边界） |

**示例**：

```java
public class Box<T extends Number> {
    private T value;

    public T getValue() { return value; }
    public void setValue(T value) { this.value = value; }
}

// 编译后（反编译结果）
public class Box {
    private Number value;  // T 被擦除为 Number

    public Number getValue() { return value; }
    public void setValue(Number value) { this.value = value; }
}
```

### 2.3 桥接方法

当泛型类继承或实现其他类时，编译器会生成**桥接方法**保证多态正确工作。

```java
public class StringList extends ArrayList<String> {
    @Override
    public String get(int index) {
        return super.get(index);
    }
}

// 编译后生成桥接方法
public class StringList extends ArrayList<String> {
    // 原方法
    public String get(int index) { ... }

    // 桥接方法（编译器生成）
    public Object get(int index) {
        return get(index);  // 调用上面的 String 版本
    }
}
```

> **为什么需要桥接方法？** `ArrayList<String>` 的 `get` 方法擦除后返回 `Object`，但 `StringList.get` 返回 `String`。JVM 通过桥接方法保证多态调用正确。

## 三、通配符

### 3.1 为什么需要通配符？

```java
List<Number> list = new ArrayList<Integer>();  // 编译错误！

// Integer 是 Number 的子类，但 List<Integer> 不是 List<Number> 的子类
```

**原因**：如果允许这样赋值，会导致类型安全问题：

```java
List<Integer> integers = new ArrayList<>();
List<Number> numbers = integers;  // 假设允许
numbers.add(3.14);  // Double 可以加入 List<Number>
Integer i = integers.get(0);  // ClassCastException！
```

### 3.2 三种通配符

| 通配符 | 含义 | 特点 |
|--------|------|------|
| `<?>` | 未知类型 | 只能读为 Object，不能写（除了 null） |
| `<? extends T>` | T 或 T 的子类 | 可读，不可写（生产者） |
| `<? super T>` | T 或 T 的父类 | 可写，读为 Object（消费者） |

### 3.3 上界通配符 `<? extends T>`

```java
// 可以读取，不能写入
List<? extends Number> list = new ArrayList<Integer>();

Number n = list.get(0);  // 可以，一定可以转为 Number
// list.add(1);  // 编译错误！不知道具体是 Integer 还是 Double

// 典型用法：从集合中读取元素
public double sum(List<? extends Number> list) {
    double total = 0;
    for (Number n : list) {
        total += n.doubleValue();
    }
    return total;
}

List<Integer> integers = Arrays.asList(1, 2, 3);
List<Double> doubles = Arrays.asList(1.5, 2.5, 3.5);
sum(integers);  // 可以调用
sum(doubles);   // 可以调用
```

### 3.4 下界通配符 `<? super T>`

```java
// 可以写入，读取类型为 Object
List<? super Integer> list = new ArrayList<Number>();

list.add(1);       // 可以，Integer 是 Integer 的子类
list.add(2);       // 可以
// list.add(3.14); // 编译错误！Double 不是 Integer 的子类

Object o = list.get(0);  // 只能读为 Object

// 典型用法：向集合中写入元素
public void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}

List<Number> numbers = new ArrayList<>();
List<Object> objects = new ArrayList<>();
addNumbers(numbers);  // 可以
addNumbers(objects);  // 可以
```

### 3.5 无界通配符 `<?>`

```java
List<?> list = new ArrayList<String>();

Object o = list.get(0);  // 只能读为 Object
// list.add("hello");    // 编译错误！不能写入非 null

list.add(null);  // null 可以，因为 null 是所有引用类型的成员

// 典型用法：不关心具体类型，只使用 Object 方法
public void printSize(List<?> list) {
    System.out.println(list.size());  // 只用到 List 的方法，不关心元素类型
}
```

## 四、PECS 原则

**Producer Extends, Consumer Super** — 生产者用 extends，消费者用 super。

| 场景 | 通配符 | 原因 |
|------|--------|------|
| 从集合**读取**数据 | `? extends T` | 保证读出来的是 T 或子类，可以安全转为 T |
| 向集合**写入**数据 | `? super T` | 保证写入的 T 可以安全存入（T 是参数类型的子类） |
| 既读又写 | 不用通配符 | 使用精确类型 `List<T>` |

**经典示例**：`Collections.copy()`

```java
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    for (int i = 0; i < src.size(); i++) {
        dest.set(i, src.get(i));  // 从 src 读（生产者），向 dest 写（消费者）
    }
}

List<Integer> src = Arrays.asList(1, 2, 3);
List<Number> dest = new ArrayList<>(Arrays.asList(0, 0, 0));
Collections.copy(dest, src);  // dest 消费 Integer，src 生产 Integer
```

## 五、泛型的限制

### 5.1 不能使用基本类型

```java
List<int> list = new ArrayList<>();  // 编译错误！

// 原因：类型擦除后变成 Object，无法存储基本类型
// 解决：使用包装类
List<Integer> list = new ArrayList<>();
```

### 5.2 不能实例化类型参数

```java
public <T> T create() {
    return new T();  // 编译错误！
}

// 原因：类型擦除后不知道 T 是什么类型
// 解决：传入 Class 对象或使用反射
public <T> T create(Class<T> clazz) throws Exception {
    return clazz.getDeclaredConstructor().newInstance();
}
```

### 5.3 不能创建泛型数组

```java
T[] array = new T[10];  // 编译错误！

// 原因：数组是协变的，类型擦除后可能导致类型安全问题
// 解决：使用 Object 数组或反射
@SuppressWarnings("unchecked")
T[] array = (T[]) new Object[10];

// 或使用反射
T[] array = (T[]) Array.newInstance(clazz, 10);
```

### 5.4 不能重载擦除后相同签名的方法

```java
public void print(List<String> list) { }
public void print(List<Integer> list) { }  // 编译错误！

// 原因：擦除后两个方法签名相同：print(List list)
```

### 5.5 异常限制

```java
public class MyException<T> extends Exception { }  // 编译错误！

// 原因：JVM 需要在运行时知道异常的确切类型
// catch 语句无法捕获泛型异常
```

## 六、类型Token与Super Type Token

### 6.1 类型擦除的困境

```java
List<String> list = new ArrayList<>();
// 运行时无法获取 String 类型信息
```

### 6.2 Class 对象作为类型Token

```java
public <T> T fromJson(String json, Class<T> clazz) {
    // 使用 clazz 知道目标类型
    return gson.fromJson(json, clazz);
}

User user = fromJson("...", User.class);
```

### 6.3 Super Type Token（超类型Token）

对于泛型类型（如 `List<User>`），Class 对象无法表示：

```java
// 无法获取 List<User>.class

// 解决方案：利用匿名内部类保留泛型信息
public abstract class TypeToken<T> {
    private final Type type;

    protected TypeToken() {
        // 获取子类的泛型参数
        this.type = ((ParameterizedType) getClass().getGenericSuperclass()).getActualTypeArguments()[0];
    }

    public Type getType() { return type; }
}

// 使用
TypeToken<List<User>> token = new TypeToken<List<User>>() {};
Type type = token.getType();  // 获得 List<User> 的类型
```

**原理**：匿名内部类在编译时会保留泛型签名信息，通过反射可以获取。

## 七、泛型与数组

### 7.1 数组的协变

```java
Object[] objects = new String[1];
objects[0] = 1;  // 编译通过，运行时 ArrayStoreException
```

数组是**协变**的：`String[]` 是 `Object[]` 的子类型。

### 7.2 泛型的不可变

```java
List<Object> list = new ArrayList<String>();  // 编译错误！
```

泛型是**不可变**的：`List<String>` 不是 `List<Object>` 的子类型。

### 7.3 为什么泛型数组不安全

```java
// 假设允许创建泛型数组
List<String>[] stringLists = new List<String>[1];
Object[] objects = stringLists;  // 数组协变，合法
objects[0] = new ArrayList<Integer>();  // 放入 Integer 列表
String s = stringLists[0].get(0);  // ClassCastException！
```

## 八、常见面试题与解答

### Q1：什么是类型擦除？有什么影响？

**答**：类型擦除是 Java 泛型的实现机制——泛型信息只在编译期存在，运行时被擦除为原始类型（Object 或边界类型）。

**影响**：
- 无法在运行时获取泛型参数类型
- 不能实例化泛型类型 `new T()`
- 不能创建泛型数组
- 重载方法不能只有泛型参数差异

---

### Q2：`List<?>`、`List<Object>`、`List` 有什么区别？

**答**：

| 类型 | 特点 |
|------|------|
| `List` | 原始类型，无类型检查，不推荐使用 |
| `List<Object>` | 精确类型，可以存任何 Object |
| `List<?>` | 未知类型，只能读为 Object，不能写（除 null） |

```java
List raw = new ArrayList();
raw.add("string");  // 编译警告
Integer i = (Integer) raw.get(0);  // 运行时异常

List<Object> objects = new ArrayList<>();
objects.add("string");
objects.add(1);  // 可以

List<?> wildcard = new ArrayList<String>();
wildcard.add("string");  // 编译错误！
Object o = wildcard.get(0);  // 可以
```

---

### Q3：什么是 PECS 原则？

**答**：PECS = Producer Extends, Consumer Super。

- **Producer（生产者）**：从集合中读取数据，使用 `? extends T`
- **Consumer（消费者）**：向集合中写入数据，使用 `? super T`

记忆口诀：**"要往外取（读）用 extends，要往里存（写）用 super"**。

---

### Q4：为什么 `List<String>` 不是 `List<Object>` 的子类型？

**答**：如果允许这种父子关系，会破坏类型安全：

```java
List<String> strings = new ArrayList<>();
List<Object> objects = strings;  // 假设允许
objects.add(new Integer(1));     // Integer 是 Object，可以加入
String s = strings.get(0);        // ClassCastException！
```

泛型是不可变的（invariant），这是为了保证类型安全。

---

### Q5：如何获取泛型的实际类型？

**答**：

1. **Class 对象**：对于非泛型类，直接使用 `User.class`

2. **TypeToken 模式**：对于泛型类，利用匿名内部类保留泛型信息

```java
TypeToken<List<User>> token = new TypeToken<List<User>>() {};
Type type = token.getType();  // List<User>
```

3. **反射获取字段/方法签名**：

```java
Field field = clazz.getDeclaredField("list");
Type genericType = field.getGenericType();  // List<User>
```

---

### Q6：泛型方法和类型通配符如何选择？

**答**：

| 场景 | 选择 |
|------|------|
| 需要在方法内部使用类型参数 | 泛型方法 `<T> void method(T t)` |
| 只用于参数/返回值类型约束 | 通配符 `void method(List<? extends Number> list)` |
| 需要多个位置使用同一类型 | 泛型方法 `<T> T method(List<T> list)` |

```java
// 泛型方法：T 在返回值和参数中保持一致
public static <T> T getFirst(List<T> list) {
    return list.isEmpty() ? null : list.get(0);
}

// 通配符：只读不关心具体类型
public static void printAll(List<?> list) {
    for (Object o : list) {
        System.out.println(o);
    }
}
```

### Q7：Kotlin 的泛型和 Java 有什么区别？

**答**：

| 特性 | Java | Kotlin |
|------|------|--------|
| 声明处型变 | 无（只有使用处通配符） | 有（`out`/`in` 关键字） |
| 星投影 | `List<?>` | `List<*>` |
| 型变声明 | `? extends T` | `out T` |
| 逆变声明 | `? super T` | `in T` |
| 具体化泛型 | 不支持 | 支持 `inline` + `reified` |

**Kotlin 的 reified 泛型**：

```kotlin
// Kotlin：可以获取泛型类型
inline fun <reified T> Gson.fromJson(json: String): T {
    return fromJson(json, T::class.java)
}

// Java：无法实现
public <T> T fromJson(String json) {
    // 无法获取 T.class
}
```
