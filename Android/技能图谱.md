## 一、Java / Kotlin 语言基础

### 1.1 Java 核心

- **JVM 内存模型**：堆、栈、方法区、程序计数器、本地方法栈的划分与作用 📄 [JVM内存模型](Java与Kotlin基础/JVM内存模型.md)
- **垃圾回收**：GC Roots 可达性分析、分代回收策略、CMS / G1 / ZGC 的区别与适用场景 📄 [垃圾回收机制](Java与Kotlin基础/垃圾回收机制.md)
- **类加载机制**：类生命周期、双亲委派模型、打破双亲委派的场景（SPI、热修复）📄 [类加载机制](Java与Kotlin基础/类加载机制.md)
- **并发编程** 📄 [并发编程与线程安全](Java与Kotlin基础/并发编程与线程安全.md)：
  - `synchronized` 原理（偏向锁 → 轻量级锁 → 重量级锁 升级过程）
  - `volatile` 语义（可见性、有序性、不保证原子性）
  - `ReentrantLock`、`ReadWriteLock`、`Condition`
  - `ConcurrentHashMap`、`CopyOnWriteArrayList` 等并发容器
  - `ThreadPoolExecutor` 七大参数、四种拒绝策略、线程池调优
  - `CAS` 与 `AQS`（AbstractQueuedSynchronizer）原理
- **泛型**：类型擦除、通配符（`<? extends T>` / `<? super T>`）、PECS 原则 📄 [泛型深度解析](Java与Kotlin基础/泛型深度解析.md)
- **反射与注解**：运行时反射、编译时注解处理（APT / KSP） 📄 [反射与注解深度解析](Java与Kotlin基础/反射与注解深度解析.md)
- **集合框架**：HashMap 底层结构（数组+链表+红黑树）、扩容机制、线程安全方案 📄 [HashMap深度解析](Java与Kotlin基础/HashMap深度解析.md)

### 1.2 Kotlin 核心

- **协程** 📄 [Kotlin协程原理](Java与Kotlin基础/Kotlin协程原理.md)：
  - 挂起函数原理（CPS 变换、状态机）
  - 协程上下文（CoroutineContext）与调度器（Dispatchers）
  - 结构化并发（CoroutineScope、SupervisorJob）
  - Flow（冷流 vs 热流、StateFlow / SharedFlow）📄 [Kotlin Flow深度解析](Java与Kotlin基础/Kotlin Flow深度解析.md)
  - 异常处理（CoroutineExceptionHandler、supervisorScope）
- **语言特性** 📄 [Kotlin语言特性深度解析](Java与Kotlin基础/Kotlin语言特性深度解析.md)：
  - 空安全设计与平台类型
  - 扩展函数与属性的本质（静态方法）
  - 密封类 / 数据类 / 值类（value class）
  - 委托属性（`by lazy`、`by Delegates.observable`）
  - 内联函数（`inline`、`noinline`、`crossinline`、`reified`）
  - DSL 构建（Lambda with receiver）

## 十、端侧 AI 与大模型

### 10.1 推理框架

- **TensorFlow Lite**：Interpreter API、Delegate 机制、模型转换（TFLite Converter）
- **ONNX Runtime Mobile**：跨框架模型标准、Execution Provider 架构
- **轻量级框架**：MNN（阿里）、NCNN（腾讯）、Paddle Lite（百度）— 适合高度定制化场景
- **MediaPipe**：Google 的端到端 ML Pipeline 方案，内置视觉/文本/音频任务

### 10.2 硬件加速

- **NNAPI**（Android 8.1+）：Android 原生的神经网络抽象层，屏蔽硬件差异
- **GPU Delegate**：基于 OpenGL ES / OpenCL 的 GPU 推理加速
- **NPU / DSP 加速**：高通 QNN（HTP）、联发科 NeuroPilot、Google Tensor TPU
- **加速策略选型**：CPU vs GPU vs NPU 的适用场景、精度与延迟权衡

### 10.3 模型优化

- **量化**：FP32 → FP16 → INT8 → INT4，训练后量化（PTQ）vs 量化感知训练（QAT）
- **剪枝与蒸馏**：结构化剪枝、知识蒸馏（大模型 → 小模型）
- **端侧 LLM 专项**：KV Cache 优化、Paged Attention、投机采样（Speculative Decoding）
- **模型格式**：GGUF、TFLite FlatBuffer、ONNX — 各格式特点与适用框架

### 10.4 Google AI Edge 生态

- **Gemini Nano**：Android 14+ 系统内置的端侧大模型，通过 AICore 系统服务调用
- **ML Kit**：开箱即用的视觉/NLP/翻译等能力，Google Play Services 动态下发模型
- **AI Edge SDK**：自定义模型部署到 Android 的统一接口

### 10.5 工程实践

- **模型打包策略**：APK 内置（体积大但离线可用）vs 动态下发（按需下载、版本管理）
- **内存管理**：大模型加载的内存规划、mmap 加载、模型分片

- **隐私合规**：端侧推理的数据不出端优势、模型知识产权保护（加密、混淆）

## 附：专精方向（按需深入）

> 以下方向不要求所有资深工程师都掌握，但在对应细分领域是核心竞争力。可根据自身职业方向选择 1-2 个深入。

### A. 音视频开发

- **采集与编码**：
  - Camera2 / CameraX API
  - MediaCodec 硬编硬解（H.264 / H.265 / AV1）
  - AudioRecord / AudioTrack、AAC / Opus 编码
- **播放与渲染**：
  - ExoPlayer / Media3：架构设计、自定义 DataSource / Renderer
  - OpenGL ES / EGL：纹理渲染、滤镜链、FBO 离屏渲染
  - Vulkan：下一代图形 API，高性能渲染场景
  - SurfaceView vs TextureView vs SurfaceControl
- **音视频处理**：
  - FFmpeg：在 Android 上的交叉编译与集成、常用命令与 API
  - MediaMuxer / MediaExtractor：封装与解封装
  - 音频混音、变速变调、降噪
- **实时通信**：
  - WebRTC：P2P 架构、ICE/STUN/TURN、Offfff/Answer 协商
  - RTMP / RTSP / HLS / DASH 流媒体协议
  - 低延迟直播方案

### B. 车载系统（Android Automotive）

- **AAOS 架构**：Android Automotive OS 与手机 Android 的区别、Car Service 层
- **VHAL**（Vehicle HAL）：车辆硬件抽象层，属性读写机制
- **Car App Library**：基于模板的车载应用开发，适配不同车机屏幕
- **系统定制**：SystemUI 定制、多屏显示（Instrument Cluster + IVI）、多用户
- **安全与合规**：驾驶分心检测、功能安全（ISO 26262）、OTA 升级
- **通信**：CAN 总线基础、车载以太网、SOA（Service-Oriented Architecture）

### C. IoT / 嵌入式 Android

- **Android Things**（已停维）→ AOSP 裁剪版用于嵌入式设备
- **低资源优化**：内存 < 1GB 场景的系统裁剪、应用精简
- **硬件交互**：GPIO / I2C / SPI / UART 通过 JNI 或 HAL 层访问
- **蓝牙 / BLE**：GATT 通信、Companion Device Manager、Nearby Connections
